# Plan: gcode parser core library (Go)

## Context

User wants a stable, reliable Go core library for parsing OrcaSlicer gcode files targeting Bambu Lab P1S printers. The library will be extended later with webhook triggers (on print start) and post-parsing hooks. The immediate need is parsing print metadata, filament info, and computing filament usage — both from parser-computed E-axis sums and from the slicer-reported footer totals (exposed separately).

Source file studied: `sample_print.gcode` — 159,439 lines, OrcaSlicer 2.3.2, single-filament PLA, 318 layers.

## Public API

### `gcode/types.go`

```go
type PrintFile struct {
    Metadata PrintMetadata
    Config   PrintConfig
    Footer   FooterSummary      // slicer-reported totals from file tail
    Layers   []LayerUsage       // per-layer computed usage (index 0 = layer 1)
}

// Computed — sum all layers up to n (1-indexed, inclusive). n=0 means all.
func (p *PrintFile) FilamentUsage(upToLayer int) (FilamentUsage, error)

type PrintMetadata struct {
    SlicerName         string
    SlicerVersion      string
    GeneratedAt        time.Time
    TotalLayers        int
    MaxZHeight         float64
    FilamentDiameter   float64   // mm
    FilamentDensity    float64   // g/cm³
    ModelPrintTime     time.Duration
    TotalEstimatedTime time.Duration
    FirstLayerTime     time.Duration
}

type PrintConfig struct {
    FilamentType    string  // PLA, PETG, ABS, …
    FilamentColour  string  // hex e.g. #000000
    FilamentVendor  string
    NozzleTemp      int     // °C
    NozzleInitialLayerTemp int
}

type FilamentUsage struct {
    LengthMM  float64
    VolumeCM3 float64
    WeightG   float64
}

type LayerUsage struct {
    Number int
    Usage  FilamentUsage
}

type FooterSummary struct {
    FilamentUsage FilamentUsage  // slicer-reported totals
    FilamentCost  float64        // slicer-reported cost
}
```

## Parser Implementation Details

**Single-pass line scanner** using `bufio.Scanner`.

**State machine sections** (tracked by comment markers):
- `HEADER_BLOCK_START` / `HEADER_BLOCK_END`
- `CONFIG_BLOCK_START` / `CONFIG_BLOCK_END`
- `EXECUTABLE_BLOCK` (layer markers + E moves)
- Post-`EXECUTABLE_BLOCK_END` → footer lines

**Header fields** (regex `; key: value`):
- `total layer number` → `Metadata.TotalLayers`
- `filament_density` → `Metadata.FilamentDensity`
- `filament_diameter` → `Metadata.FilamentDiameter`
- `max_z_height` → `Metadata.MaxZHeight`
- `generated by` line → slicer name, version, timestamp
- `model printing time` / `total estimated time` → durations
- `estimated first layer printing time` → duration

**Config fields** (regex `; key = value`):
- `filament_type`, `filament_colour`, `filament_vendor`
- `nozzle_temperature`, `nozzle_temperature_initial_layer`

**Layer boundary**: `; layer num/total_layer_count: N/318`
- On each new layer marker: flush accumulated E for previous layer into `[]LayerUsage`
- Reset per-layer accumulator

**E-axis accumulation** (M83 relative mode throughout):
- Match `G1 ... E<float>` — only positive values (skip retractions)
- Convert: `volume_mm³ = Σ(E) × π × (diameter/2)²`
- `volume_cm3 = volume_mm³ / 1000`
- `weight_g = volume_cm3 × density`
- Length directly from `Σ(E)` mm

**Footer** (after `EXECUTABLE_BLOCK_END`):
- `; filament used [mm] = <float>`
- `; filament used [cm3] = <float>`
- `; filament used [g] = <float>`
- `; filament cost = <float>`

## `FilamentUsage(upToLayer int)` method

- `upToLayer == 0` → sum all layers
- `upToLayer > 0` → sum `Layers[0..upToLayer-1]`
- Returns error if `upToLayer > len(Layers)`
- Each `LayerUsage.Usage` holds pre-computed per-layer values; method just sums

## Future Hook Points (not implemented now)

`WithLayerHook` is wired into the parser: after each layer is finalised, call `opts.onLayerComplete(layer, usage)` if set. This is the extension point for the planned webhook-on-print-start feature.
